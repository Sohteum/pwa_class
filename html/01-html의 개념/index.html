html Hypertext Markup Language

웹문서의 DOM구조를 설계하는 마크업 언어를 의미

html문서의 특징

  • 요소(element), 태그(tag), 속성(attribute)이 존재
  • Node(parent, children, decendent, siblings)가 존재
  • node는 정해진 구조를 준수해야 함

시멘틱요소

구획요소(기능별로 영역을 나눔)

  • main : 웹문서의 주요내용
  • section, article : 제목과 내용을 포함하는 주제를 다루는 영역
  • header : 사이트의 정체성을 출력하는 영역
  • footer : 사이트의 보조정보를 출력하는 영역(회사정보 등)
  • aside : 사이트의 부수적인 정보를 출력하는 영역(광고 혹은 사이드바)
  • nav : 메뉴기능을 제공하는 영역
  • div : 레이아웃 영역(시멘틱적인 의미X,기능적영역)

(다양한 시멘틱요소)논리요소(logical element)

  • em : 중요함
  • strong, mark : 강조(시청각적), 마크도 강조
  • small : 보조적의미(예를들면 한글단어옆에 보조적으로 영어로)
  • i : 구분 다름의 목적, (아이콘이나 같은단어다른뜻, 어드레스태그의 항목
  • blockquote : 인용(글)
  • cite : 인용처(홈페이지 등)
  • span : 의미가 없음(태그로의 확장, 디자인목적)
  • time : 시간
  • adress : 주소, 전화, 등
  • abbr : 약어(MS사에서 blablabla했다)
  • dfn : 용어(MS사는 microsoft이다)

물리적 요소(physical element,시각적요소),사용자제&css사용

  • b : 두꺼운 글자
  • u : 밑줄
  • strike : 취소선
  • sup : 윗첨자2,필요시 사용
  • sub : 아래첨자 2,필요시 사용
