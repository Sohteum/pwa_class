1. indexOf

  • 배열 안에서 특정값이 몇번째 인덱스에 있는지를 알아냄:무조건 숫자가 나옴
  • 값이 존재하지 않으면 -1(음수)를 반환
  • 동일한 값이 여러개 있더라도 첫번째 인덱스만 반환 후 종료 즉 값이 겹치면 먼저것만 나오고 끝남.대표적으로 아이디같은거 검색할때
  • 객체는 검사를 못함 그때부터는 파인드인덱스를써야함
      ['가', '나', '다']

2. findIndex

  • 배열 값으로 객체나 배열이 중첩되어있을 경우 인덱스 검색
  • 콜백함수 내에서의 매개변수(파라미터.v)와 조건문과 리턴문을 사용
      [
      {'이름': '홍길동', '나이':'35'},
      {'이름':'아무개', '나이':'22'},
      {'이름':'무명씨', '나이':'24'},
      ]

3. findIndex

      [
      {'이름': '홍길동', '나이':'35'},
      {'이름':'아무개', '나이':'22'},
      {'이름':'무명씨', '나이':'24'},
      ]

4. forEach

  • 배열의 길이만큼 순환
  • 콜백함수 내에서의 매개변수와 조건문과 리턴문을 사용
      [
      {'이름': '홍길동' '나이':'35', '성별':'남'},
      {'이름':'아무개', '나이':'22','성별':'여'},
      {'이름':'무명씨', '나이':'24','성별':'남'},
      { '이름': '영이씨', '나이': '24', '성별': '여' },
      ]
