syntax rule

법

html 유효성(validator) 검사(kldp,w3c/https://validator.kldp.org/)를 이용해 확인해 볼것

문법규칙

  • inline요소의 자식으로는 block요소는 허락되지 않는다 예외 : a는 모든 요소의 부모로 허락됨(인터렉티브 제외!확인!)
  • heading요소(h1~6), p요소의 자식으로는 인라인(인라인블럭)계열레벨만 허락된다
  • interactive요소간에는 부모자식관계가 허락되지 않는다(사용자가 클릭하면 바뀌는 폼요소,앵커같은)
  • figcaption의 자식으로는 a요소가 허락되지 않는다(웹사이트 만들면서 다룰예정)
  • ul,ol의 자식요소로는 li만 허락된다
  • li의 자식으로는 모든 요소가 허락된다(ul도가능함/스타일같은건안돼...헤드에만 쓸수있어,스크립트는 다됨)
  • dl의 자식으로는 dt,dd요소만 허락된다
  • style,meta,title요소는 head의 자식요소로만 허락된다
  • img요소는 alt속성(보조텍스트)을 가져야 한다

권장사항

매너
  • heading요소(제목)는 텍스트를 포함하길 권장
  • h1과 main요소는 웹문서에 한개씩 포함하길 권장
  • main,section,article요소는 heading요소(제목)를 한개 이상 포함하길 권장
  • heading요소는 순차적으로 적용하길 권장(h1의 번호 순으로,지키기는 어렵)
  • title값은 page마다 다르게 출력하길 권장
  • WAI-ARIA규칙을 준수하길 권장,알고는 있자(aoa11y웹접근성만연구하는유튭)
